# 定时任务首页 Figma 还原设计

## 目标

仅调整定时任务首页前端样式，使其尽量贴近 Figma 节点 `958:10855`，不改动后端接口、任务数据结构、创建编辑弹窗行为和运行记录逻辑。

## 范围

- 重做首页头部标题区与操作按钮样式。
- 重做顶部 4 张统计卡片样式与排版。
- 重做任务列表容器、表头、行高、列内信息层级和状态区域。
- 保留“新建任务 / 编辑 / 删除 / 立即运行 / 历史记录 / 启停切换”现有交互。
- 兼顾桌面优先的响应式表现，移动端允许降级但不破版。

## 非目标

- 不改动后端 `scheduledtasks` 服务。
- 不新增批量勾选、批量操作、筛选、分页。
- 不重写创建任务和运行记录弹窗。
- 不调整左侧导航样式。

## Figma 还原要点

### 页面头部

- 左侧为两行信息：
  - 标题：`定时任务`
  - 副标题：`通过定时任务自动化执行AI任务`
- 右侧为两个操作按钮：
  - 次按钮：`刷新`
  - 主按钮：`添加任务`
- 整体留白比当前实现更宽，按钮高度约 36px，圆角 8px。

### 统计卡片

- 4 张白底卡片横向排列。
- 卡片包含浅灰色圆形图标容器、24px 数字、14px 标签。
- 圆角 16px，细描边，轻阴影。
- 文案顺序保持：
  - 任务总数
  - 运行中
  - 已暂停
  - 失败

### 列表区

- 外层为白底大卡片，8px 圆角，边框色偏浅灰。
- 表头与正文采用 Figma 中较轻的分隔线，不使用过重底色。
- 列顺序保持现有：
  - 任务
  - 执行时间
  - 上次运行
  - 下次运行
  - 状态
  - 操作
- 任务列为两行：
  - 主标题：任务名称
  - 次信息：prompt 摘要
- 执行时间列为两行：
  - 主信息：调度描述
  - 次信息：最近执行时间或说明
- 状态列为开关 + 文案组合，尽量贴近设计稿的浅灰/深色切换观感。
- 操作列保持三点菜单，不新增 inline 按钮组。
