# 运行记录弹窗宽度与时间展示设计

**目标:** 调整定时任务运行记录弹窗的视觉排版，使整体宽度增加 200px，并让左侧运行时间展示保持单行且更紧凑。

**范围:**
- 修改运行记录弹窗最大宽度。
- 调整左侧列表时间文案的字号和换行行为。

**非目标:**
- 不调整运行记录的数据结构、交互逻辑和接口调用。
- 不修改右侧会话预览组件。

**设计决策:**
- 将弹窗 `sm:max-w-[1400px]` 调整为 `sm:max-w-[1600px]`，直接满足“整体宽度增加 200px”。
- 将左侧时间文本从 `text-sm` 调整为 `text-xs`。
- 为时间文本增加 `whitespace-nowrap`，确保时间不换行。

**风险与验证:**
- 风险较低，仅影响局部样式类名。
- 通过静态检查变更内容，确认宽度与时间样式类名已更新。
