# 定时任务空态设计

## 目标

当定时任务列表没有数据时，页面按照 Figma 节点 `982:12302` 展示整页空态，核心标题为“暂无定时任务”，并保留现有创建任务交互。

## 范围

- 保留页面顶部标题、副标题、刷新按钮和添加任务按钮
- 保留统计卡片区域
- 当列表为空且非加载态时，使用页面级空态替换列表区域
- 空态内展示图标、标题、说明文案和“新建任务”按钮
- 空态按钮复用现有 `openCreateDialog` 逻辑

## 非目标

- 不改动后端接口和 `useScheduledTasks` 数据获取逻辑
- 不改动创建任务弹窗字段和提交流程
- 不改动有数据时的表格结构和交互

## 方案

### 结构

- 在 `ScheduledTasksPage.vue` 新增 `hasTasks` 计算属性
- `loading` 时继续显示加载态
- `!loading && !hasTasks` 时渲染页面级空态
- `hasTasks` 时继续渲染 `TaskTable`

### 组件职责

- `ScheduledTasksPage.vue` 负责页面级空态判断与交互绑定
- `TaskTable.vue` 只负责有数据时的表格展示，不再承担空列表文案展示

### 文案

- 标题使用现有 `scheduledTasks.empty`
- 说明文案补充独立 i18n key，内容与设计图一致
- 空态按钮文案使用现有“新建任务”语义，避免和右上角“添加任务”混淆

### 样式

- 空态区域位于统计卡片下方，整体水平居中
- 图标容器为浅灰背景、圆角 8px 的 40x40 容器
- 标题使用 16px 中黑字重
- 说明文案使用 14px 灰字、限制宽度并居中
- 主按钮沿用页面现有深色主按钮视觉

## 风险与取舍

- 页面级空态更贴近设计稿，但需要把 `TaskTable` 的空列表逻辑上移
- 当前仓库未发现前端单测基础设施；若不补测试框架，则只能通过类型检查和构建验证实现正确性

## 验证

- 空列表时显示页面级空态
- 点击空态“新建任务”按钮可打开现有创建任务弹窗
- 有列表数据时仍显示 `TaskTable`
- 执行 `npx vue-tsc --noEmit`，如环境允许再执行 `npm run build`
