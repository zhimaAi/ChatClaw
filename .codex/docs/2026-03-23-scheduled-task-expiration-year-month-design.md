# 定时任务到期时间年月快速切换设计

**背景**

当前新建/编辑定时任务弹窗中的到期时间使用自定义日历面板，只支持通过左右箭头逐月翻页。在目标日期距离当前月份较远时，切换成本较高。

**目标**

在不改变“最终仍需选择具体日期”的前提下，为到期时间面板增加年份和月份的快速切换能力，提升远距离跳转效率。

**方案**

- 保持 `form.expiresAtDate` 继续作为唯一数据源，格式仍为 `YYYY-MM-DD`。
- 在 `TaskFormContent.vue` 的到期时间日历头部增加两个原生下拉：
  - 年份下拉：围绕当前可见月份提供连续年份范围。
  - 月份下拉：提供 `1-12 月`。
- 用户切换年份或月份时，只更新 `visibleExpirationMonth`，不直接写入表单。
- 用户点击某一天后，才真正更新 `form.expiresAtDate` 并关闭面板。
- 保留现有左右切月按钮、今天、清空、只读态、已过期提示和关闭行为。

**边界**

- 不修改后端接口、DTO、校验和提交逻辑。
- 不引入新的日期组件库。
- 不支持仅选择“年-月”后直接提交。

**验证**

- 为新增的年月切换逻辑补前端测试，覆盖年份范围和切换结果。
- 运行针对性的前端测试与 `vue-tsc` 校验。
